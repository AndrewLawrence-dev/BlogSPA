Interviews

How would you diagnose/fix a front-end performance problem?

The first step is to determine whether the problem really is a front-end problem. It may be a back-end issue. I would generate a performance profile of the application (using the 'Performance' tab, for example, in Chrome Developer Tools) and check the Network requests to see if any network requests are being made, how long each takes to get a response, the datasize of the request, etc... If a request is taking a long time to complete, there's a problem with the back-end or with whatever server(s) the network call is being made to.

If there aren't any requests that are taking a long time, we then need to isolate the problem within the front-end code base. You can continue to drill into the performance profile while looking for any events that are triggered before/after the noticeable performance problem. This will help locate the source of the issue. Loops, DOM manipulations, rendering, etc... are common places of performance issues. You can use "performance.now()".

I would also consider using "performance.now()" statements around sections of code to further track the run-time of an operation(s).

How do you resolve customer issues?

The most important thing is to get a good understanding of what they expect to see and what they actually see. We must start out on the same page having the same understanding of what the issue is (and isn't).

If it's a coding issue, I would start by getting as much information about the issue as I can (such as error messages, what they were doing when the issue occurred, etc...). I would then examine any logs to see if an error (exception) has been logged in the database.

For more general issues, it's important to let the customer know that you're committed to seeing the issue through and that you're not looking for a quick fix to relieve yourself. I would assure them that I will look into the issue and that they are free to contact me with any information that they may find.

How do you handle errors on the server (back-end)?

I would make sure exception handling is implemented using 'try-catch' blocks. I would also make sure that errors are being logged in plain English with detailed information. If an error occurs, the user must be informed in plain English as well.

I started my first long-term software development job as a web developer for WingWeb Consulting. I worked with the owner of the company. For PPS, I am responsible for the entire codebase. I communicate directly with the business owner to help him get what he really needs (not necessarily only what he asks for). I oversee his website that he uses as an internal eCommerce site to track inventory and orders.

I am now returning to the C#/.NET world. Venturing out taught me new perspectives and possibilities for solving problems.
